자바스크립트의 오리엔테이션

웹브라우저
  -브라우저를 코드를 통해 제어한다(이런게 프로그래밍적인 제어라고 한다)
  자바스크립트라는 프로그래밍 언어는 웹브라우저라는 형태의 분류되는 소프트웨어들을 프로그래밍적으로 제어하기 위한 언어
  자바스크립트와 웹브라우저는 뗄라야 뗄 수 없는사이

탈웹브라우저
  -탈이라고 붙어있는 이유는 자바스크립트가 더이상 웹브라우저를 제어하는 용도로만 사용되고 있지 않다.
  자바스크립트 용도
    -웹서버를 동작하기 위한 도구로 사용된다. 이런걸 server side script 라고 한다
    (대표적 기술 : node.js)
    기본적으로 자바스크립트가 만들어진 이유
    웹서버 --->  웹브라우저
    자바스크립트 기술 개발이 발달되면서
    자바스크립트를 웹서버에서도 사용할 수 있도록 하는 기술들이 등장했다.
    웹서버에 사용되는 대표적 기능 (php,java,python)
    node.js를 사용하게 되면 웹서버를 제어하기위한 javascript를 쓸 수 있다.
    웹전체에서 자바스크립트 영향력이 확장되고있다

탈웹
  - 웹에서 사용되던 자바스크립트가 웹 바깥쪽 기술에서 사용되기 시작했다
  대표적 사례 google apps script 이다
  구글 엑셀
  중요한 점은 자바스크립트가 웹브라우저를 제어하는 도구로 끝나는 게 아니라 자바스크립트를 배우면 여러분야에서 적용해 사용할 수 있다.

언어
  - 의사소통을 위한 약속

환경이란?
  - 탈웹, 탈웹브라우저 다양한 분야에서 사용하기 때문에 언어와 언어가 동작하는 환경을 분리해서 생각하면 큰 효능이 있다.

언어를 사용하는 대상
  - javascript('hello world'); 자바스크립트 프로그래밍 문법, 자바스크립트가 동작하는 언어
  alert (웹브라우저) - 자바스크립트가 동작하는 환경
  write (node.js) - 자바스크립트가 동작하는 환경
  msgBox (spreadsheet) - 자바스크립트가 동작하는 환경

정리

사용자 입장에서 웹브라우저,node.js,스프레드시트 제어한다는 것은 마치 버튼을 클릭하는 것고 비슷하다
개발자가 된다는 의미는 코드를 통해서 제품들을 제어한다.
